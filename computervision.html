Portfolio Assignment 8. Hand position and distance


  • In this exercise, we learnt about poseNet, an algorithm that detects body key points and can be used for creations on the screen. We were asked to visualize and play with wrist key points and the distance between them in an interesting way.

  • At the beginning, even with tutorial, I was confused by the way the program works and the naming, and error that shows up when I try to add a red nose to the nose key point. so I used console.log to help me understand more about its mechanisms. What I learnt here is

  • 1) it takes time for poseNet model to load and the draw function can execute before poseNet even finished loading, which explains why the editor stops as it cannot determine location of nose when it receives no pose data.

  • 2) poses is an array that contains all key points and skeleton information. poses[0] is an object inside the array that contains both pose and skeleton information; and “pose” further breaks down that information into 17 keypoints that contains x, y coordinates information of that pose. By simplifying poses[0].pose into just “pose” and write as pose.nose.x, This means we want to access “the value of the x coordinate of the nose key point that belongs to “pose” category and is inside an array called poses that is detected by poseNet”.

  • Finally my nose showed up and I added circles to my wrists and use for loops to make access other key points in the array to draw small circles and eventually strokes for skeleton.





  • Once I got more familiarised with using distance between key points to create variables that I can use to adjust the sizing of shapes with my hand movements, I experimented with ellipse and then a heart shape function and it became fun.
